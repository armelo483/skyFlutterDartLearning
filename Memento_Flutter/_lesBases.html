Architecture projet flutter:

  • android: du code généré automatiquement pour la plateforme android, donc à ne pas toucher, car généré depuis le dossier lib
  • ios: idem pour la plateforme ios 
  • lib: c'est où se trouve le code de l'appli (dart et flutter)
  • .packages:  fichier généré automatiquement par pub(pub est le gestionnaire de package de flutter)
  • .iml: Fichier représentant l'architecture du projet généré automatiquement par androidStudio 
  • pubspec.yaml: Fichier où on met les librairies pour les installer dans le projet 
  • pubspec.lock: Fichier contenant le numero de version de chaque librairie du projet
  • test: dossier pour tester l'application en y écrivant des scénarii de tests
  • lib/main.dart: Point d'entrée de l'appli, un peu comme le main.c en langage "C"
